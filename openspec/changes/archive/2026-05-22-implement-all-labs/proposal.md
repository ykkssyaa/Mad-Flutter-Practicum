# Proposal: Единое корневое Flutter-приложение для всех лабораторных работ

Коротко: цель — собрать все требования лабораторных работ в одно полноценное Flutter-приложение в корне репозитория, ориентированное на Windows desktop, без дублирования отдельных приложений по `lab_1`…`lab_6`.

## Capabilities

- `root-flutter-app` — собрать единое приложение в корне проекта на основе требований из `lab_1`…`lab_6`
- `windows-desktop-support` — обеспечить запуск и удобную работу приложения на Windows desktop

Почему это нужно:
- В результате получится один цельный, поддерживаемый проект, а не набор почти одинаковых приложений.
- Проще демонстрировать, тестировать и сопровождать результат по всем лабораторным работам.
- Можно переиспользовать общие компоненты, тему, навигацию, данные и локализацию без копирования кода между папками.

Область работ (ссылки на требования):
- ЛР #1: `lab_1/"Lab 1. Widget layout.md"` — требования к базовой верстке и виджетам
- ЛР #2: `lab_2/"Lab 2. State management.md"` — требования к управлению состоянием
- ЛР #3: `lab_3/"Lab 3. Navigation.md"` — требования к навигации и экранному потоку
- ЛР #4: `lab_4/"Lab 4. Network.md"` — требования к сетевому слою и загрузке данных
- ЛР #5: `lab_5/"Lab 5. Local data.md"` — требования к локальному хранению и кэшу
- ЛР #6: `lab_6/"Lab 6. Localization.md"` — требования к локализации и пользовательским настройкам

Критерии готовности изменений:
- В корне репозитория существует одно запускаемое Flutter-приложение.
- Приложение корректно запускается на Windows desktop и имеет понятную структуру для дальнейшего расширения.
- Все ключевые требования из markdown-описаний ЛР учтены в единой кодовой базе.
- Набор тестов и базовые проверки (`flutter analyze`, `flutter test`) проходят на корневом проекте.
- `README.md` содержит краткую инструкцию запуска и проверки именно для единого приложения.

Ограничения и допущения:
- Подзадачи ссылаются на существующие markdown-описания лабораторных работ; детализированно пересказывать их в артефактах не нужно.
- Учебные тексты `Lab *.md` не меняются; меняются только корневая реализация, конфигурации и вспомогательная документация.

Результат:
- Папка `openspec/changes/implement-all-labs/` с обновлёнными артефактами `proposal.md`, `design.md`, `tasks.md`, описывающими единое корневое Windows-приложение.
